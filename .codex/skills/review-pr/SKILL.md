# review-pr

## Purpose
Run a structured pull request review that emphasizes correctness, scope control, and maintainability.

## Workflow
1. Verify PR scope matches issue/task intent.
2. Review architecture boundaries and package ownership.
3. Check tests, type safety, and failure-mode coverage.
4. Evaluate readability, API stability, and migration impact.
5. Produce findings grouped by severity with actionable suggestions.

## Outputs
- Review summary (approve/request changes).
- Prioritized findings and recommended fixes.
- Risk notes and suggested follow-up tasks.
