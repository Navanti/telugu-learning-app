# feature-implementation

## Purpose
Guide a focused feature delivery workflow from clarified requirements to implementation-ready tasks.

## Workflow
1. Confirm scope, impacted packages/apps, and non-goals.
2. Draft acceptance criteria and edge cases before coding.
3. Identify owner agent(s) and split work into small tasks.
4. Implement in the owned package boundaries.
5. Add/update tests and run validation commands.
6. Prepare concise PR notes with risks and follow-ups.

## Outputs
- Task checklist with owners.
- Implementation notes and impacted paths.
- Test evidence summary.
